# 更新记录
## v3.0.8.7 (2026-06-10)
- **视频源支持分章下载**：在添加和管理视频源时可以单独开启“分章下载”，下载后会按 B 站章节直接生成独立媒体文件，并复用现有 NFO、封面和缩略图生成流程，不再保留完整原视频，也不会额外创建章节子文件夹；启用“多P视频使用 Season 文件夹结构”时，分章视频会按多P的 Season 目录和多P分页命名模板生成。
- **分章下载可以和音频模式共存**：开启“仅下载音频”或“仅保留 M4A”时不会再强制清掉分章下载；仅保留 M4A 时只保留拆分后的章节音频，不再生成章节 NFO、封面和背景图。
- **下载选项显示补齐**：视频源列表和视频详情页会同时显示“仅下载音频”“仅保留 M4A”“平铺目录”“分章下载”等已启用选项，修复多个“下载”标签只能显示一个的问题。

## v3.0.8.6 (2026-06-04)
- **NFO 时间选择不再假生效**：设置里选“收藏时间”或“发布时间”后，后面生成出来的 NFO 会真的按这个选择写时间，不会再出现页面显示保存了、文件里还是旧时间的情况。
- **NFO 会写入更具体的时间**：以前很多媒体库只能看到日期，现在 NFO 里会多写一份带“几点几分几秒”的时间，按时间排序和识别会更准。
- **多 P 视频的总 NFO 也会跟着刷新**：切换收藏时间/发布时间后，不只是单 P 和分集会更新，多 P 视频目录里的总 NFO 也会一起重做，不会再残留上一次的时间。
- **AI 改名后的文件名更干净**：AI 返回的名字如果还有中文括号、多余空格、问号、冒号、结尾横杠这些问题，程序会再兜底清一遍，减少文件名看着乱或者 Windows 不好保存的问题。

## v3.0.8.5 (2026-05-28)
- **自定义源名称不再被改回接口名**：修复合集、收藏夹和 UP 主投稿源在扫描时，会把用户手动改过的显示名称重新覆盖成 B 站接口返回名称的问题；现在已有源会保留本地自定义名称，只同步保存路径等必要信息。
- **文件命名可以先预览**：设置文件名模板时，页面会直接预览最后生成出来的视频文件名，每行一个 `.mp4`，不用保存后再猜会下载成什么名字。
- **改文件名模板后不会把视频路径记成 NFO**：以前文件名从标题改成 BV 号这类模板后，重命名时可能把 `.nfo` 当成视频文件记进数据库，导致后面继续处理文件时出错；现在只会把真正的视频文件记进去。
- **DeepSeek 改名不再空响应误报**：AI 重命名使用 DeepSeek 付费 API 时，会关闭不需要的思考模式，避免模型只返回推理内容却没有给出文件名 JSON，导致批次报“解析 JSON 数组失败”。
- **自动续登确认更稳**：刷新 B 站 Cookie 时，会固定用刷新前的 `ac_time_value` 完成确认，再保存接口返回的新凭据，避免新旧刷新口令混用导致自动续登失败。

## v3.0.8.4 (2026-05-26)
- **单P文件名模板重新生效**：修复 v3.0.8 后单P视频即使配置了“单P视频文件名模板”，下载出来仍只用标题命名的问题；现在会按设置里的日期、BV号、标题截断等模板生成视频、NFO、弹幕、字幕和封面文件名。
- **应用文件名模板不再乱追加目录后缀**：修复手动应用文件名模板时，已经正确的父目录被误判为同名冲突、反复追加“完整发布时间-BV号”后缀的问题；真正同标题视频的目录去重规则仍会保留。

## v3.0.8.3 (2026-05-16)
- **自动续登不再乱报过期**：每天 1 点会直接跑完整的 B 站登录刷新流程；如果只是网络断了、请求超时，或者 B 站临时抽风，不会再提示“登录过期、请重新扫码”，而是提示“凭据自动刷新暂时失败”。
- **设置页可以手动测续登**：B 站凭据那里新增“测试刷新”和“强制刷新”，可以直接看刷新卡在哪一步；强制刷新会真的跑完整个续登流程，方便排查到底是网络、B 站返回，还是登录信息本身有问题。
- **续登时间按 B 站为准**：刷新登录信息时会尽量用 B 站给的时间，不再完全依赖本机时间，减少电脑时间不准导致刷新失败的情况。
- **添加源搜索先走原来的方式**：添加视频源时，默认仍先用原来的搜索接口；如果这个接口被 B 站拦住或返回内容不对，再换网页端的新搜索方式兜底。
- **搜索会带上登录信息和指纹**：搜索请求会带上已经配置好的 B 站登录信息和浏览器指纹，减少像无痕浏览那样没有指纹导致被 412 拦住的问题；如果带了以后还被拦，会明确提示是 B 站风控。
- **UP 主搜索结果能拉全**：搜索“小王纸”这类 UP 主时，不再只显示第一页，会按 B 站返回的实际页数继续加载，把后面的结果也拉出来。

## v3.0.8.2 (2026-05-13)
- **合集分页临时超时重试**：获取 B 站合集分页时如果遇到临时超时，会自动短暂重试，减少合集源因为单次 `-504` 服务超时就中断整轮扫描的问题。
- **登录失效通知补齐**：检测到 B 站登录状态过期或未登录，并且自动刷新 Cookie 失败时，会通过已配置的通知渠道提醒处理，避免只写日志导致长期无法发现。
- **凭据每日自动刷新**：程序会在每天 1 点检查并刷新 B 站 Cookie，正常过期时会自动保存新凭据到数据库，降低扫描开始时才发现登录失效的概率。
- **刷新失败原因更清楚**：Cookie 刷新链路会区分缺少凭据、刷新页面取不到 `refresh_csrf`、刷新接口失败等情况；如果手动退出登录或清除登录状态，仍需要重新扫码登录或更新整套凭据。

## v3.0.8.1 (2026-05-09)
- **签名信息获取更稳**：获取播放地址前需要用到的 B 站签名信息如果偶发请求失败，会自动再试几次，减少因为短暂网络波动导致整轮下载失败的问题。
- **坏 CDN 地址自动换路**：下载时如果某个 B 站 CDN 节点返回的证书和地址对不上，会先跳过这个坏地址继续试备用地址；一批地址都不行时，会重新获取播放地址再试，减少单个异常 CDN 节点拖垮整个视频下载的问题。

## v3.0.8 (2026-04-28)
- **签名信息获取更耐抖**：扫描开始时如果网络、DNS 或代理短暂连不上，会先短暂重试几次，不再因为一瞬间连接失败就直接跳过整轮扫描、留下错误日志。
- **超长文件名截断**：默认生成的文件名和目录名会按 100 字节截断，减少长中文标题在生成弹幕、字幕、封面等配套文件时超过文件系统限制的问题。
- **弹幕更新顺序优化**：视频源还有未下载、未修复或待重试的视频时，会先处理这些下载任务，不再一开局把旧视频的到期弹幕一起塞进主流程；等该源下载处理完后，下一轮再按规则更新旧弹幕，减少大 UP 主源触发风控的概率。
- **添加视频源页面修复**：添加 UP 主投稿、合集或收藏夹时，搜索结果、关注列表、合集列表和收藏夹列表不会再同时残留在页面上，选择目标后会自动收起无关列表。
- **同名视频目录冲突修复**：同一个 UP 主下出现多个同标题视频时，不再把后下载的视频挤进同一个裸标题目录；发生冲突的视频会使用“标题-完整发布时间-BV号”作为目录名，避免不同视频文件混在一起。
- **单P文件名更稳定**：如果视频已经单独占用一个目录，目录内的视频文件会保留标题文件名；只有多个视频确实共用同一个目录时，才会在文件名里加完整发布时间和 BV 号。
- **重设路径保留 AI 改名目录**：AI 重命名父目录后，再重设视频源路径会保留当前 AI 改过的目录名，只把目录整体迁到新路径下，不再退回默认文件夹格式。
- **重设路径迁移修复**：重设视频源路径时会把实际移动后的目录同步写回数据库，减少目录被追加 `_1`、`_2` 后页面路径和本地文件不一致的问题。
- **发布时间格式统一**：命名、通知、投稿选择和视频源最新时间等位置的发布时间统一使用完整时间戳，例如 `20260428170830`，避免只显示日期导致同一天视频难以区分。
- **前端显示同步**：设置页默认时间格式和分页命名提示已同步更新，视频源管理页、投稿选择页会按完整时间戳展示相关发布时间。

## v3.0.7 (2026-04-23)
- **视频详情来源标签**：视频详情页会显示该视频来自哪个视频源，方便确认它是从收藏夹、合集、UP主投稿、稍后再看还是番剧入库的。
- **入库概览修正**：把“最新入库”和“最近处理”分开，真正新加的视频才进最新入库，补弹幕或重处理的老视频只进最近处理，避免误以为新视频丢了。
- **DeepSeek 重命名修复**：修复批量重命名时偶发漏掉返回开头、导致整批解析失败的问题；默认模型也已换成 DeepSeek 新模型，设置页提示同步更新。
- **视频封面本地兜底**：视频管理页远程封面打不开时，会继续尝试读取已经下载到本地的封面；B 站返回失效占位时，也不会把已有标题和封面改成空信息，历史占位标题会用已下载分页名兜底显示。
- **删除队列卡住修复**：修复删除视频或视频源后，队列状态可能一直停在“处理中”、后续删除只能排队不继续执行的问题。
- **删除残留任务清理**：程序启动恢复删除任务时，如果目标视频源已经不存在，会继续检查并清理残留的视频和分页记录；删除本地文件后只沿本次删除路径向上回收空目录，不再全量扫描媒体根目录。
- **暂停后删除视频源修复**：修复下载任务刚暂停、后台还在收尾时立即删除视频源，可能导致本地文件夹被晚到的下载任务重新生成、网页端却已经搜不到的问题；现在会等当前任务真正结束后再执行删除。

## v3.0.6 (2026-04-20)
- **平铺目录重设路径修复**：修复稍后观看等视频源在平铺目录模式下重设保存路径时，只在目标位置留下空文件夹、真实视频和配套文件没有一起搬走的问题；现在会把视频、本地封面、NFO、弹幕、字幕一起迁到新位置。
- **简介页显示更干净**：修复媒体库里简介区域把原始视频链接和一堆标签文字直接显示出来的问题，现在详情页看起来更清爽。
- **仅自己可见的视频不再反复重试**：有些被作者设成“仅自己可见”或已经看不到的视频，现在会更早跳过，不会一轮里反复报错、重复尝试。
- **通知没配完整时不再刷失败日志**：如果你把 Webhook 选成了自定义方式，但内容没填完整，程序会先跳过这类通知，不再一遍遍提示发送失败；手动点测试时还是会明确告诉你哪里没配好。
- **NFO 分类标签设置**：关掉这个设置后，只会影响后面新生成的 NFO；以前已经生成好的 NFO 不会被重做，也不会被改掉。
